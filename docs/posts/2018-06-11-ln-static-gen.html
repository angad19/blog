Ln: A simple, extensible static site generator

Angad Singh | 2018-06-11 16:21

I've started building a lot more projects. I wanted to write about problems I ran into, tools I used and just general stuff. I've tried a lot of ways to blog, and almost all of them has some or the other problem which made them harder to use for me. Jekyll felt overkill, I don't know React so Gatsby or anything React based was nto an option. I'm not great at Python, so most of the Python-based options were out too. I also wanted a nice weekend project to work on after my SSAT was over.

Writing my own generator was fun. It wasn't very difficult, but challenging enough to keep me interested. I wrote Ln in JavaScript, because I'm know JavaScript well enough to write useable software.

Ln is a simple and extensible static site generator. It is made up of three components - posts.json, plugins and config.js. posts.json is read and an array of post objects is created. An ln plugin is just a function that takes an array of post objects and the config as params and returns a promise which resolves with an array of posts. So, for example, you could build a plugin that reads all org files in a directory, generates post objects from them and adds them to the posts array. After awaiting all the plugins, the modified array of post objects is written to posts.json. config.js exports an object whose properties can be accessed by all plugins.

You can then run the index script to run the build function which calls all the plugins.
